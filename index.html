AI Agents - Business Operating Leverage

AI Agents & Workflow Automation

Ethos: build AI agents the way you would build a serious operating system for your life or business: visible, measured, permissioned, reviewable, and useful every day. The win is not replacing people with a chatbot. The win is removing repetitive drag, keeping context alive, catching exceptions early, and giving humans more leverage.

Most stock agent installs are impressive demos but weak operating layers. They can answer, code, and use tools, but they often lack durable memory, shared visibility, scheduled follow-up, Telegram accountability, watchdogs, dashboards, and a review loop. Our current stack is designed to close that gap.

OpenClawLocal automation and control layer: terminal UI, gateway, Telegram channels, browser/control tasks, plugins, memory, sessions, and dashboard access.
HermesPlanning and execution layer: coding, repo edits, scheduled jobs, memory retrieval, local APIs, research, Telegram updates, verification, and dashboard access.
Master ControlVisibility layer: projects, live services, scheduled tasks, all sessions, memory, Obsidian notes, and agent engagement from one cockpit.
Human Review LayerTrust layer: approval gates, evidence, audit trails, fail-closed workflows, exception surfacing, and handoff back to a person.

Concerns and pitfalls to respect

  • Silent failure: agents can look busy while missing the real business outcome. Track inputs, outputs, receipts, and verification.
  • Tool overreach: never let an agent delete, spend, message customers, or change systems without explicit permission and rollback paths.
  • Context rot: long chats drift. Use memory deliberately, compact sessions, and keep source-of-truth docs clean.
  • Model cost and rate limits: long tool loops burn tokens. Monitor usage and pick the cheapest model that can reason well enough for the job.
  • Browser automation fragility: APIs first, browser automation second, desktop/manual fallback last.
  • Trust gap: if a human cannot see what happened, why it happened, and what changed, the workflow is not production-ready.

Why this can become a cheat code

When agents are tracked and monitored properly, they become a compounding leverage system: they remember decisions, chase loose ends, prepare work, run checks, produce evidence, and keep pushing while you sleep or focus elsewhere. That is the real cheat code — not one magic prompt, but a monitored agent stack that keeps life and business workflows moving.

Build principles

  • Start with one painful repeatable workflow, not a vague “automate everything” goal.
  • Give the agent bounded tools, test data, and a clear success definition.
  • Make every workflow observable: logs, Telegram updates, dashboard state, and final receipts.
  • Keep humans on judgment, customer trust, finance, safety, legal, quality, and edge cases.
  • Scale only after the workflow proves it saves time without creating hidden risk.

Guide 1 - What is an AI agent?

What is an AI Agent?

Short answer: an AI agent is an AI assistant that can use tools to get work done, not just answer questions.

Normal chatbot vs AI agent

Chatbot
  • Answers questions.
  • Mostly text in / text out.
  • You do the clicking, copying, saving, and checking.
AI Agent
  • Can break a goal into steps.
  • Can use tools like files, browser, terminal, search, memory, and calendars.
  • Can keep working until a task is complete, then show evidence.

What can an AI agent do?

📄 DocumentsSummarise, rewrite, make checklists, create drafts.
🔎 ResearchSearch, compare options, build a simple report.
🧾 AdminPrepare emails, follow-up lists, meeting notes, SOPs.
🧠 MemoryRemember preferences, projects, and reusable workflows.
🛠️ ToolsRead files, create files, check websites, run safe commands.
📊 BusinessSurface exceptions, reduce duplicate entry, help with reporting.

Real workflow examples

Email → CRM → Welcome → Follow-upRead a new enquiry email, extract name/contact/service/suburb, check for duplicates, enter the lead into a CRM, send an approved welcome email, schedule follow-ups until the customer replies, then hand the conversation to a human with a clean summary.
Quote prep assistantRead job notes, photos, customer requirements, and pricing rules; prepare a draft quote pack; flag missing information; notify Telegram for approval; only send after human sign-off.
Operations exception radarCheck dashboards and inboxes each morning, find stale leads, overdue jobs, missing payments, or review risks, then post a ranked action list with links and evidence.
Knowledge-base maintainerWatch repeated staff questions, draft SOP updates, link source documents, ask for approval, and publish a cleaner internal answer so the next person does not ask again.

The power is the chain: read context → update systems → communicate → schedule next action → monitor for reply → escalate to a human when judgment or trust matters.

What it should not do without you

  • Spend money, delete files, send public messages, or change business systems without approval.
  • Make legal, medical, financial, safety, or employment decisions on its own.
  • Replace human judgment where customer trust, risk, quality, or approvals matter.

Guide 2 - Install your first local AI agent

Install Your First Local AI Agent

Goal: get a real agent running locally, connect it to a model, and talk to it through Telegram so testing feels like messaging a helpful assistant instead of operating a developer tool.

Cost + tokens intro: AI tools charge or limit you by tokens — chunks of text sent in and generated back. Long chats, big files, and repeated tool use burn tokens quickly. At the moment, GPT / ChatGPT OAuth is usually the cheapest beginner path for a good balance of reasoning and usability when the tool offers it. Start with a ChatGPT Plus account. Move to Pro only once you are actually hitting rate limits or using too many tokens for that plan. Pricing and limits change, so always check the provider page before relying on it for business-critical work.

Before the Web UI: terminal setup you must complete

The Web UI/dashboard is the easy part. First you need the local agent configured in Terminal: model login, local workspace, safe permissions, messaging gateway, and health checks.

Hermes required terminal flow

  1. Run setup wizard:
    hermes setup
    Choose local/default workspace options unless you know you need something custom.
  2. Select model/provider: when prompted, choose GPT / ChatGPT OAuth if available. Log in through the browser window it opens. Start with ChatGPT Plus; upgrade to Pro once rate limits/token use genuinely block you.
  3. Confirm model:
    hermes model
    Use this if you need to change provider/model later.
  4. Configure Telegram/messaging gateway:
    hermes gateway setup
    Paste your BotFather token only into this local setup prompt. Authorise your own Telegram user/chat when prompted.
  5. Install/start gateway service:
    hermes gateway install
hermes gateway start
hermes gateway status
    If you are on WSL/Docker/Termux, foreground mode may be easier for first testing:
    hermes gateway run
  6. Health check:
    hermes doctor
hermes status
    Fix red errors before opening the dashboard.
  7. Only after that, open Web UI:
    hermes dashboard
    For a server/WSL machine where you do not want auto-open:
    hermes dashboard --host 127.0.0.1 --port 9119 --no-open

OpenClaw required terminal flow

  1. Run first setup:
    openclaw setup --wizard --mode local
    Choose a local workspace. Beginners should keep the default unless they have a reason to move it.
  2. Configure model credentials:
    openclaw configure --section model
    When prompted, choose GPT / ChatGPT OAuth if available. If not, use the provider/API key option the wizard supports. Do not paste tokens into chats.
  3. Configure workspace + safe defaults:
    openclaw configure --section workspace
openclaw configure --section gateway
    Keep the gateway local/loopback for first testing unless you know how you want to expose it.
  4. Configure Telegram channel:
    openclaw configure --section channels
    Select Telegram, paste your BotFather token, and authorise your test user/group/channel.
  5. Start or install gateway:
    openclaw gateway run
    For a background service after it works:
    openclaw gateway install
openclaw gateway start
openclaw gateway status
  6. Health checks before Web UI:
    openclaw doctor
openclaw status
openclaw gateway status
    Fix red errors before opening the dashboard.
  7. Only after that, open Web UI:
    openclaw dashboard
    For terminal-only/server testing:
    openclaw dashboard --no-open

Hermes Agent on Mac — complete beginner path

  1. Prepare: open Terminal. Install Apple command-line tools if macOS asks.
  2. Install Hermes:
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
  3. Restart Terminal so your shell sees the new hermes command.
  4. Run local setup:
    hermes setup
hermes doctor
  5. Model login: when setup asks for a model/provider, choose GPT / ChatGPT OAuth if available. Otherwise use a supported API key/provider you already have access to. Do not paste keys into public chats.
  6. Pick a safe working folder: make a test folder on Desktop and ask Hermes to use that first, not your whole computer.
  7. First local test:
    hermes chat -q "I am new to Hermes. Explain what you can do, then create a safe beginner checklist for me."
  8. Telegram path: follow the BotFather section below, then run setup again if needed and enable the messaging gateway.

Hermes Agent on Windows — recommended WSL Ubuntu path

  1. Open PowerShell as Administrator.
  2. Install Ubuntu under WSL:
    wsl --install -d Ubuntu
  3. Restart if Windows asks. Open Ubuntu from Start Menu and create a Linux username/password.
  4. Update Ubuntu:
    sudo apt update && sudo apt upgrade -y
  5. Install Hermes inside Ubuntu:
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
  6. Close and reopen Ubuntu, then run:
    hermes setup
hermes doctor
  7. Model login: choose GPT / ChatGPT OAuth if offered for the cheapest beginner balance of reasoning. Start with Plus, move to Pro only after rate limits become a real blocker.
  8. Working files: your Windows Desktop is usually under /mnt/c/Users/YOURNAME/Desktop. For first tests, create a small folder and keep Hermes scoped there.

Hermes Agent on Linux — local install

  1. Open Terminal and make sure curl exists:
    sudo apt update
sudo apt install -y curl git
  2. Install Hermes:
    curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
  3. Reload shell:
    exec $SHELL -l
  4. Run setup and checks:
    hermes setup
hermes doctor
  5. Model login: choose GPT / ChatGPT OAuth if available, or a supported provider/API key. Keep keys in the setup/config flow, not in prompts.
  6. First local test:
    hermes chat -q "Create a safe starter plan for learning local AI agents. Ask before touching files."

OpenClaw on Mac — complete beginner path

  1. Install Node.js 22+. Easiest Mac path is Homebrew:
    brew install node
    If you do not have Homebrew, install Node.js from nodejs.org.
  2. Check Node:
    node --version
npm --version
    OpenClaw currently requires Node 22.14+.
  3. Install OpenClaw globally:
    npm install -g openclaw
  4. Run local onboarding/config:
    openclaw setup
openclaw configure
openclaw doctor
  5. Model login: choose GPT / ChatGPT OAuth if the wizard offers it. Otherwise add your provider/API key through the config wizard. Start with Plus; upgrade to Pro only when rate limits genuinely slow your work.
  6. First local chat:
    openclaw chat
    Then ask: Explain what you can safely do on this computer and ask before taking actions.

Telegram + BotFather setup — main testing comms path

  1. Open Telegram and search for @BotFather. Confirm it has Telegram’s verified checkmark.
  2. Send /newbot.
  3. Choose a friendly display name, e.g. My First Agent.
  4. Choose a unique username ending in bot, e.g. my_first_agent_123_bot.
  5. BotFather gives you an HTTP API token. Copy it once and store it safely. Treat it like a password.
  6. Open your new bot in Telegram and press Start, or send /start.
  7. For channel/group testing: create a private Telegram group or channel, add your bot, and make it admin only if it needs to post there. Keep beginner testing private.
  8. Run your agent setup again and choose Telegram/messaging gateway when prompted:
    hermes setup
# or
openclaw configure
  9. Paste the BotFather token only into the official local setup/config prompt.
  10. Start/enable the gateway when the tool asks. Then send a simple message to your bot: Say hello and explain what you can do safely.

Beginner safety checklist

  • Start in a test folder, not your whole Desktop/Documents drive.
  • Do not give access to money, customer systems, or private business data on day one.
  • Ask the agent to show a plan before edits or sends.
  • Keep Telegram groups private while testing.
  • If a command fails, run hermes doctor or openclaw doctor and read the red lines.
